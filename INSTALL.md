# 📘 Инструкция по установке и запуску проекта

Полное пошаговое руководство для дипломного проекта  
**Автоматизированная система учёта посещаемости АВПК**

---

## 🎯 Что вам нужно сделать (всего 4 шага)

1. ✅ Создать проект в Firebase (бесплатно, ~10 минут)
2. ✅ Вставить конфигурацию Firebase в код
3. ✅ Запустить проект локально или загрузить на GitHub Pages
4. ✅ Создать первого администратора

---

## 📋 ШАГ 1. Создание проекта в Firebase

### 1.1. Регистрация и создание проекта

1. Откройте сайт **https://console.firebase.google.com**
2. Войдите через свой аккаунт Google (или зарегистрируйтесь)
3. Нажмите кнопку **«Создать проект»** (Add project)
4. Введите название проекта, например: `avpk-attendance`
5. На шаге Google Analytics — можно отключить (необязательно)
6. Нажмите **«Создать проект»** и дождитесь окончания создания (~1 минута)

### 1.2. Включение Authentication (вход по email/паролю)

1. В левом меню Firebase Console выберите **Build → Authentication**
2. Нажмите **«Начать»** (Get started)
3. Перейдите на вкладку **«Sign-in method»**
4. Найдите **«Email/Password»** и нажмите на него
5. Включите переключатель **«Enable»** и нажмите **«Save»**

### 1.3. Создание базы данных Firestore

1. В левом меню выберите **Build → Firestore Database**
2. Нажмите **«Create database»** (Создать базу данных)
3. Выберите режим: **«Start in test mode»** (Тестовый режим) — для разработки
   - ⚠️ Тестовый режим работает 30 дней. Этого хватит для защиты диплома.
4. Выберите регион (например, `eur3` или `europe-west`)
5. Нажмите **«Enable»**

### 1.4. Получение ключей конфигурации

1. Нажмите на **шестерёнку** ⚙️ в левом верхнем углу → **«Project settings»**
2. Прокрутите вниз до раздела **«Your apps»**
3. Нажмите на иконку **`</>`** (Web app)
4. Введите название приложения, например: `АВПК — посещаемость`
5. **НЕ** ставьте галочку «Set up Firebase Hosting» (пока что)
6. Нажмите **«Register app»**
7. Вы увидите код примерно такого вида:

```javascript
const firebaseConfig = {
  apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXX",
  authDomain: "avpk-attendance.firebaseapp.com",
  projectId: "avpk-attendance",
  storageBucket: "avpk-attendance.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789012:web:abcdef1234567890"
};
```

**📌 Скопируйте этот объект — он понадобится на следующем шаге!**

## 📋 ШАГ 2. Вставка конфигурации в код

1. Откройте файл **`js/firebase-config.js`** в любом редакторе (VS Code, Notepad++, и т.д.)
2. Найдите блок:

```javascript
const firebaseConfig = {
    apiKey: "ВАШ_API_KEY",
    authDomain: "ваш-проект.firebaseapp.com",
    // ...
};
```

3. **Замените** все значения на свои из Firebase Console (которые скопировали ранее)
4. Сохраните файл

---

## 📋 ШАГ 3. Запуск проекта

### Вариант А: Запуск локально (для тестирования)

Просто откройте `index.html` через **локальный сервер**. Через `file://` не будет работать из-за модулей JavaScript.

**Способ 1 — через VS Code (рекомендуется):**
1. Установите расширение **Live Server** в VS Code
2. Откройте папку проекта в VS Code
3. Правый клик на `index.html` → **«Open with Live Server»**
4. Проект откроется в браузере

**Способ 2 — через Python (если установлен):**
```bash
# В папке проекта откройте терминал и выполните:
python -m http.server 8000

# Затем откройте в браузере:
# http://localhost:8000
```

**Способ 3 — через Node.js (если установлен):**
```bash
npx serve
```

### Вариант Б: Бесплатный хостинг на GitHub Pages

Это лучший вариант для защиты диплома — проект будет доступен по ссылке из любой точки мира.

#### Шаг 3.1. Регистрация на GitHub
1. Откройте **https://github.com** и зарегистрируйтесь
2. Подтвердите email

#### Шаг 3.2. Создание репозитория
1. Нажмите **«+»** в правом верхнем углу → **«New repository»**
2. Введите название, например: `avpk-attendance`
3. Выберите **Public** (публичный)
4. **НЕ** ставьте галочку «Add a README file»
5. Нажмите **«Create repository»**

#### Шаг 3.3. Загрузка проекта на GitHub

**Способ А — через веб-интерфейс (самый простой):**
1. На странице нового репозитория нажмите **«uploading an existing file»**
2. Перетащите все файлы проекта (или выберите через кнопку)
3. Внизу нажмите **«Commit changes»**

**Способ Б — через командную строку (если установлен Git):**

```bash
# В папке проекта откройте терминал:
git init
git add .
git commit -m "Дипломный проект: система учёта посещаемости"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/avpk-attendance.git
git push -u origin main
```

#### Шаг 3.4. Включение GitHub Pages

1. В вашем репозитории зайдите в **Settings** (вкладка вверху)
2. В левом меню выберите **Pages**
3. В разделе **«Source»** выберите ветку **`main`** и папку **`/ (root)`**
4. Нажмите **«Save»**
5. Подождите 1–2 минуты
6. Обновите страницу — увидите ссылку вида:
   ```
   https://ВАШ_ЛОГИН.github.io/avpk-attendance/
   ```

#### Шаг 3.5. Разрешение домена в Firebase

⚠️ **ВАЖНО!** После публикации на GitHub Pages нужно добавить домен в список разрешённых:

1. Откройте Firebase Console
2. **Authentication → Settings → Authorized domains**
3. Нажмите **«Add domain»** и добавьте: `ВАШ_ЛОГИН.github.io`
4. Нажмите **«Add»**

## 📋 ШАГ 4. Создание администратора и заполнение системы

### 4.1. Первый вход — регистрация админа

1. Откройте сайт (локально или на GitHub Pages)
2. На странице входа нажмите **«Зарегистрироваться»**
3. Заполните форму:
   - ФИО: например, «Администратор Системы»
   - Email: ваш email
   - Пароль: придумайте свой пароль (минимум 6 символов)
   - **Роль: Администратор**
4. Нажмите **«Зарегистрироваться»**
5. Вы автоматически войдёте в систему

### 4.2. Заполнение базы данных

Войдя как администратор, добавьте:

1. **Предметы** (Меню → «Предметы»): математика, программирование и т.д.
2. **Группы** (Меню → «Группы»): ИС-21, ПР-22 и т.д.
3. **Преподаватели**: они регистрируются сами через форму регистрации с ролью «Преподаватель»
4. **Студенты**: они тоже регистрируются сами с ролью «Студент», а вы потом привязываете их к группам через раздел «Студенты»
5. **Расписание** (Меню → «Расписание»): создавайте занятия

### 4.3. Отметка посещаемости

1. Войдите как преподаватель или администратор
2. Перейдите в «Посещаемость» или «Расписание»
3. Выберите занятие
4. Отметьте статус для каждого студента
5. Нажмите «Сохранить»

## 🔒 Настройка правил безопасности Firestore (для production)

⚠️ **После защиты диплома** (или сразу, если хотите большей безопасности) измените правила безопасности:

1. Firebase Console → **Firestore Database → Rules**
2. Замените содержимое на:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Разрешаем чтение всем авторизованным пользователям
    match /{document=**} {
      allow read: if request.auth != null;
    }
    
    // Запись разрешена в зависимости от ролей
    match /users/{userId} {
      allow create: if request.auth != null;
      allow update: if request.auth.uid == userId 
                    || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    
    match /groups/{id} {
      allow write: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    
    match /students/{id} {
      allow write: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    
    match /subjects/{id} {
      allow write: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    
    match /schedule/{id} {
      allow write: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role in ['admin', 'teacher'];
    }
    
    match /attendance/{id} {
      allow write: if get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role in ['admin', 'teacher'];
    }
  }
}
```

3. Нажмите **«Publish»**

## ❓ Решение возможных проблем

### Проблема: «Firebase не инициализирован»
**Решение:** Проверьте, что вы правильно вставили `firebaseConfig` в файл `js/firebase-config.js`.

### Проблема: При входе ошибка «auth/invalid-credential»
**Решение:** Проверьте правильность email и пароля. Если забыли — создайте нового пользователя.

### Проблема: «Permission denied» при загрузке данных
**Решение:** Убедитесь, что Firestore запущен в **тестовом режиме**, либо настройте правила безопасности (см. выше).

### Проблема: На GitHub Pages не работает вход
**Решение:** Добавьте домен `ВАШ_ЛОГИН.github.io` в **Authentication → Settings → Authorized domains** в Firebase.

### Проблема: Открываю index.html напрямую — пустой экран
**Решение:** Нужен локальный сервер (Live Server в VS Code или `python -m http.server`). Через `file://` модули JavaScript не загружаются.
